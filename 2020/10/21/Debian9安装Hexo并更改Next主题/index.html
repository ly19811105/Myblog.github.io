Debian9安装Hexo并更改Next主题

Posted on 2020-10-21

Debain 9 安装 Hexo 并配置到Github

把过程和坑记录一下

检查SSH 文件是否存在,详细请参看官方文档

bash ls -al ~/.ssh

如果这里root/.ssh目录没有 id_rsa 和 id_rsa.pub 文件用下面命令生成一个

生成SSH key 文件

bash ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

检查ssh-agent

bash eval "$(ssh-agent -s)"

把SSH key 加入ssh-agent

bash ssh-add ~/.ssh/id_rsa

配置Github SSH 验证

bash cat ~/.ssh/id_rsa.pub

登录Gighub网站，在Setting里添加从id_rsa.Pub 文件里copy的Key到SSH登录设置里,并用以下命令验证

bash ssh -T git@github.com

收到 Hi username! You’ve successfully authenticated, but GitHub 这段表示SSH设置成功

1. 安装必要依赖 Curl sudo git

apt-get install curl -y
apt-get install sudo -y
apt-get install git -y

2. 更改NodJS 源

curl -sL https://deb.nodesource.com/setup_14.x | sudo bash -

3. 安装Nodjss

apt install nodejs -y

4. 更改NPM 源

npm config set registry https://registry.npm.taobao.org

5. 安装Hexo

cd ~
mkdir hexo
cd ~/hexo
npm install hexo-cli -g
hexo init Myblog

‘如果这里有Path错误提示，按照提示创建文件夹 Mkdir

cd blog
npm install
hexo g # 或者hexo generate
hexo s # 或者hexo server，可以在http://LocalIP:4000/ 查看

用浏览器检查是否安装成功

6. 要能成功部署，需要提前安装一个扩展：

$ npm install he:xo-deployer-git --save

7. 使用hexo deploy部署

hexo deploy可以部署到很多平台，具体可以参考这个链接. 如果部署到github，需要在配置文件_config.xml中作如下修改：

deploy:
    type: git
    repo: git@github.com:xxx(your username).github.io.git
    branch: master

然后在命令行中执行

hexo d

8. 使用git命令行部署

使用hexo depoly确实很方便，但是有的时候会出现一些bug，所以个人还是更喜欢直接用Git命令来完成部署。
clone github repo 进入你的blog文件夹，然后clone你的GitHub repo

git clone https://github.com/xxx(your username).github.io.git .deploy/xxx(your username).github.io

上一行代码的意思是将我们之前创建的repo克隆到本地的一个叫做.deploy的新建的文件夹用于存放克隆的代码。
将下面的代码直接封装成sh，每次需要部署的时候直接bash执行就行。
具体每行代码的意思见注释。

cp -R public/* .deploy/xxx(your username).github.io
#这一行是复制blog下面的public中的网页代码到我们Git的本地文件夹
cd .deploy/xxx(your username).github.io
#进入这个本地的Git文件夹
git add .
#加载此文件夹下面所有的文件
git commit -m “update”
#提交代码
git push 
#push到GitHub上

到这里你的个人博客基本框架搭建起来了，这里根据你自己所选的主题配置的不同，你的个性化设置也可以不一样，有很多的配置，现在不熟悉也没有关系，可以慢慢的摸索，后面我还会分享一些个人搭建过程中的小小心得经验

9. 更改主题

跳转到你的Hexo主目录，你可以看到 node_modules, source, themes和其它目录

cd hexo
ls
_config.yml  node_modules  package.json  public  scaffolds  source  themes

从Github 下载主题

mkdir themes/next
curl -s https://api.github.com/repos/theme-next/hexo-theme-next/releases/latest | grep tarball_url | cut -d '"' -f 4 | wget -i - -O- | tar -zx -C themes/next --strip-components=1

配置主题生效

在 hexo 根目录下 的配置文件_config.yml里设置主题:

cd ~/hexo
theme: next
